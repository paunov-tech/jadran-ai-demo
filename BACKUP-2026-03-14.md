## KEY ARCHITECTURE

### Payment Flow
```
User → Paywall OR Landing Banner → startCheckout() / goToStripe()
  → POST /api/checkout → Stripe Checkout (hosted)
  → Stripe redirect → /ai?payment=success&session_id=...
  → POST /api/verify → { paid: true }
  → setPremium(true) + setStep("chat") + SuccessModal
  → Webhook: POST /api/webhook → receipt_email + Firestore write
```

### Session Persistence (new this session)
```
localStorage keys:
  jadran_region        — persisted on every setRegion()
  jadran_travelMode    — persisted on every setTravelMode()
  jadran_session       — temp: saved before Stripe, deleted after return
  jadran_ai_premium    — "1" while active, removed on expiry
  jadran_premium_plan  — JSON: {plan, days, region, expiresAt, ...}
  jadran_device_id     — permanent device identifier (jd_...)
  jadran_utm           — UTM params with 30-day TTL
  jadran_msg_count     — trial message counter
  jadran_invited_by    — referral inviter device ID
```

### Premium Auto-Skip Logic
```
step initializer:
  1. Check jadran_ai_premium === "1"
  2. Check expiresAt > Date.now() (NEW: expiry check)
  3. Check jadran_region exists
  4. Check jadran_travelMode exists
  5. Check NOT payment redirect URL
  → All true? step = "chat" (skip setup)
  → Otherwise: step = "setup"
```

### Post-Payment Fallback Chain
```
Region: savedSession.region → localStorage jadran_region → "split"
Mode:   savedSession.travelMode → localStorage jadran_travelMode → "apartment"
→ ALWAYS enters chat, never stuck on setup
```
